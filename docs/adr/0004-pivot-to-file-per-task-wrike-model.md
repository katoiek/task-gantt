# ファイル単位タスクの Wrike 風モデルへ転換 / Pivot to a file-per-task, Wrike-style model

ADR-0001〜0003 を廃止し、ClickUp / Wrike を Obsidian で再現するモデルへ転換する。

**何を / What:**
- **1 ファイル = 1 タスク**。スケジュール情報（開始・終了・ステータス・担当・先行タスク）は各ノートの**フロントマター**が源泉。ノート**本文（Body）**がタスク説明。
- 設定した**親フォルダ直下のサブフォルダ＝グループ**、その中の `.md` ＝タスク。
- リボンアイコンで専用 Gantt ビューを開く。レイアウトは `[フォルダ/ナビ | タスク表(中央) | タイムライン(右)]`（Wrike 形式）。表とタイムラインは**行が揃う**。
- 描画は**自前の表＋SVG タイムライン**（frappe-gantt は左の表を持てないため不採用）。
- タスククリックで右から**詳細パネル**（日付・ステータス・担当・Body）がスライドイン。
- バーのドラッグで各ファイルのフロントマター `start`/`end` を書き戻し。依存は矢印表示・違反は警告のみ（自動スケジュールなし）。

**なぜ / Why:** 実際に DSL 方式を動かした結果、ユーザーの本当の要件は「ClickUp/Wrike のようなタスク管理 UI を Obsidian で」だった。タスクを独立ノートにすると Obsidian のリンク・検索・本文編集とそのまま統合でき、Wrike 風の表＋タイムライン体験に直結する。

**廃止する決定 / Superseded:** ADR-0001（DSL を源泉）, ADR-0002（YAML DSL）, ADR-0003（frappe-gantt）。
